# AEO (Answer Engine Optimization) 정리

## 1. AEO란?

**AEO (Answer Engine Optimization, 답변 엔진 최적화)** 는 사용자의 질문에 대해 검색엔진이나 AI가 직접 "답변"을 제시할 때, 내 콘텐츠가 그 답변으로 선택되도록 최적화하는 전략이다.

기존의 SEO(Search Engine Optimization)가 "검색 결과 목록에서 상위에 노출되는 것"을 목표로 했다면, AEO는 "AI가 직접 말하는 답변에 내 정보가 포함되는 것"을 목표로 한다.

---

## 2. AEO가 등장한 배경

- ChatGPT, Claude, Gemini, Perplexity 같은 **생성형 AI 검색**의 확산
- 구글의 **AI Overview (AI 요약 답변)**, 네이버 **Cue:** 등 AI 답변형 검색 도입
- 사용자가 더 이상 "10개 링크 중에 고르지 않고", AI가 정리해 준 답을 그대로 받는 경향
- → **"링크 클릭"이 아니라 "답변에 인용되기"가 새로운 트래픽/노출 기준**이 됨

---

## 3. SEO vs AEO 비교

| 구분 | SEO | AEO |
|------|-----|-----|
| 목표 | 검색 결과 상위 노출 | AI 답변에 인용/선택 |
| 사용자 행동 | 링크 클릭 | 답변 그대로 소비 |
| 핵심 단위 | 키워드, 백링크 | 질문, 명확한 답변 문장 |
| 평가 주체 | 검색 알고리즘 | LLM (생성형 AI) |
| 콘텐츠 형태 | 긴 글, 키워드 밀도 | 짧고 정확한 답, 구조화된 정보 |

---

## 4. AEO 핵심 전략

### (1) 질문 중심으로 콘텐츠 작성
- "○○이란 무엇인가?", "○○ 하는 방법은?" 같이 사용자가 실제로 묻는 형태로 제목/소제목 구성
- FAQ 섹션 적극 활용

### (2) 답을 먼저, 설명은 나중에
- 첫 1~2문장에서 질문에 대한 **결론(요약 답)** 을 명확하게 제시
- AI는 보통 첫 문단에서 답을 추출함

### (3) 구조화된 데이터 (Structured Data)
- **Schema.org** 마크업 활용 (FAQPage, HowTo, Article 등)
- AI가 콘텐츠의 의미를 이해하기 쉽게 표시

### (4) 신뢰성과 출처
- **E-E-A-T** (Experience, Expertise, Authoritativeness, Trustworthiness)
- 작성자 정보, 출처 링크, 최신 업데이트 날짜 명시
- AI는 신뢰도 높은 사이트를 우선 인용

### (5) 짧고 명확한 문장
- AI가 그대로 인용하기 좋은 **단답형 + 보충 설명** 구조
- 한 문단 = 한 가지 정보

### (6) 다양한 형식 제공
- 텍스트 + 표 + 리스트 + 이미지 + 비디오
- AI는 표나 리스트에서 정보를 잘 추출함

---

## 5. AEO 실전 체크리스트

- [ ] 콘텐츠가 사용자의 **실제 질문**으로 시작하는가?
- [ ] 첫 문장에 **답이 요약**되어 있는가?
- [ ] FAQ 섹션이 있는가?
- [ ] 표/리스트로 정리된 부분이 있는가?
- [ ] Schema 마크업이 적용되어 있는가?
- [ ] 작성자/출처/날짜가 명확한가?
- [ ] 모바일에서도 잘 읽히는가?
- [ ] 핵심 키워드 + 자연어 질문 둘 다 커버하는가?

---

## 6. 한 줄 요약

> **SEO는 "검색되는 글"을 만드는 것, AEO는 "AI가 인용하고 싶은 답"을 만드는 것.**

앞으로의 콘텐츠는 사람보다 **AI가 먼저 읽고**, 그 AI가 사람에게 전달하는 구조이므로,
**"AI가 이해하기 쉽고, 그대로 인용하고 싶은 형태"** 로 만드는 것이 핵심이다.
